# Huashu-Design Visual Reference

Distilled from the huashu-design skill. Use these principles for cover image direction and inline image suggestions in WeChat article HTML.

## 默认策略：纯文字，不配图

一篇好的公众号文章不需要图片撑场面。**默认输出不包含任何配图。**

只有用户明确要求配图时，才在 HTML 里插入文字占位块（见下方「配图占位块」）。

## Core Design Philosophy

1. **从已有上下文出发** — 好设计从已有内容长出来。视频本身就是上下文：关键帧、说话人的表情、展示的幻灯片、演示的产品。

2. **资产 > 规范** — 品牌被认出来靠的是 logo、产品图、界面截图，不是色值和字体。在文章配图里，「品牌」就是视频内容本身。

3. **一个细节做到 120%，其他做到 80%** — 选一张 hero 图精做，不要每张图都平均用力。

4. **宁缺毋滥** — 一张 6 分的图对可信度的伤害比没有图更大。宁可少放，不要凑数。

## 封面图方向

### 原则

- 封面是文章在信息流里的第一印象
- 它应该做出文章能兑现的承诺
- 封面上文字必须在缩略图尺寸可读（公众号信息流约 100×100px）
- 风格跟内容调性匹配：
  - **观点/分析** → 粗字体、高对比、极简
  - **教程/方法** → 干净排版、步骤可视化、编号
  - **故事/案例** → 人物元素、暖色调、情绪化
  - **数据/研究** → 图表、数字、结构感

### 构图规则

- 留 30%+ 负空间给文字
- 关键视觉信息不要放在边缘（信息流预览会裁切）
- 标题在封面上的位置：上 1/3 或居中，视觉占其余
- 颜色：最多 2-3 个主色，一个强调色给 CTA

### 字体

- 中文标题：粗体、无衬线（思源黑体、苹方、或系统默认）
- 正文不要用手写/书法字体
- 对比度必须足够手机阅读
- 字号层级：主标题 > 副标题 > 标签

### 避免

- 不增加信息的通用 stock 图
- 纯装饰性的图
- 文字叠在花哨背景上（缩略图看不清）
- 「AI 味」太重的生成图（太光滑、太完美、恐怖谷）
- 一张封面上超过 3 种字体
- 杂乱——封面做 5 件事等于一件都没做好

## 配图占位块格式

只有当用户明确要求配图时，才在 HTML 里插入。**只用文字描述，不用本地文件路径：**

```html
<p><strong>配图建议：</strong>这里放一张「结构化提问 vs 模糊提问」的对比截图，左边输入模糊，右边输入具体，输出质量差异一眼可见。</p>
```

## 内文配图设计

当用户说「要配图」「帮我设计几张图」时，**不要给封面图的排版变体**——用户要的是跟文章内容对应的、各自独立的插图。每张图应该对应文章里一个具体的观点或比喻。

### 三个验证过的配图模式

以下三个模式来自真实实战，覆盖了大部分公众号文章配图场景：

#### 模式 1：对比图（左右分屏）

适合场景：文章里有「A vs B」「会 vs 不会」「之前 vs 之后」的对比论点。

做法：
- 左右各占一半，中间用分隔线或标签隔开
- 左边 = 错误/低效的做法（配色收敛、灰调、小字）
- 右边 = 正确/高效的做法（配色明确、大字、强调色）
- 分隔线上标注「同一个工具」「同样的场景」等上下文
- 底部一行小字总结核心信息

实例：`illus-1-question-gap.html`——左边「帮我写个登录功能」→ 烂结果，右边「实现 refresh token rotation 登录」→ 生产级结果，中间标「同一个 AI 工具」。

#### 模式 2：词典/卡片网格

适合场景：文章提出一个「收集/整理/分类」的概念（提示词词典、方法论工具箱、知识体系）。

做法：
- 3×N 或 4×2 网格排列色块卡片
- 每张卡片 = 一个领域/类别，包含图标 + 名称 + 数量
- 最后一张卡片用虚线框 + 半透明，表示「待填补」
- 卡片颜色各不同但明度/饱和度一致，整体和谐
- 顶部一行标题 + 底部一行说明

实例：`illus-2-prompt-dictionary.html`——8 张色块卡片（编程调试/写作文案/数据分析/设计视觉/沟通谈判/决策思考/学习阅读/你的领域），像词典格子。

#### 模式 3：比喻/场景插图

适合场景：文章里有一个强比喻（F1 赛车、老中医望闻问切、登山）。

做法：
- 用 CSS 几何形状构建场景，不依赖外部图片
- 把比喻对象放在画面中央（赛车、山、放大镜等）
- 关键数字或状态用 UI 元素呈现（档位表、刻度尺、标签）
- 顶部一句核心文案（大号粗体），底部状态说明
- 颜色：背景暖白/浅灰，主体用强调色，对比强

实例：`illus-3-f1-metaphor.html`——深灰赛道上一台朱红 F1 静止不动，档位表显示 1/8，顶部大字「你开的不是卡罗拉，是一台 F1 赛车」。

### 配图设计规范

- 尺寸：750×500 或等比缩放，适配公众号正文宽度（~375px 逻辑像素）
- 颜色：跟封面共享同一套色板，保持文章视觉一致
- 文字在图中：必须足够大，手机上看不用放大就能读
- 不用外部图片素材，全部用 CSS 几何形状 + 文字完成
- 每张图底部标注一句话核心信息
- 不要 AI 味的霓虹渐变、电路板纹路、机器人插图

### 封面 vs 配图：区别

| | 封面 | 内文配图 |
|------|------|---------|
| 目的 | 信息流里让人停下来点开 | 辅助理解文章里的具体观点 |
| 跟文章关系 | 概括全文，一句承诺 | 对应一个具体段落或比喻 |
| 数量 | 1 张 | 2-4 张，每张讲不同的事 |
| 文字量 | 标题 + 最多一句辅助文字 | 可以有较多说明文字 |
| 变体 | 给排版变体让用户选 | 每张是完全不同的画面 |

## 提示词格式（供后续生成或找设计师用）

```
任务：{封面 / 配图 / 示意图}
内容：{图要传达什么}
风格：{视觉风格参考}
色调：{配色方向}
构图：{构图说明}
文字：{画面上要出现的文字}
参考：{可参考的关键帧或参考图}
避免：{不要做的事}
```

## Placeholder Strategy

没有好图时：
- 用诚实的占位块（文字描述该放什么）
- 绝不用烂图凑数
- 「先空着」比「放张烂图」专业一百倍
