# WeChat HTML Format Reference

公众号编辑器对粘贴的 HTML 有一定兼容性。以下规则确保复制过去后结构干净，不需要二次调整。

## HTML 标签白名单

只用这些标签：

| 标签 | 用途 |
|------|------|
| `<h1>` | 文章标题（一篇只用一个） |
| `<h2>` | 小节标题 |
| `<p>` | 正文段落 |
| `<blockquote>` | 引用/关键句/摘要 |
| `<strong>` | 重点句（每段最多 1-2 处） |
| `<section>` | 摘要区（仅用于导语） |
| `<br>` | 段落内换行（少用） |

## 绝对不用

- `<div>` — 公众号编辑器处理不一致
- `<img src="/Users/...">` — 本地路径粘贴后无效
- `<script>` — 会被过滤
- `<style>` — 内联样式也会被部分过滤，用标签语义就够了
- `<span>` — 不用内联样式就不需要 span
- `<table>` — 手机端阅读体验差
- `<ul>` `<ol>` — 可以保留，但公众号编辑器可能重置缩进
- `class` / `id` — 粘贴后会被清除，写了也没用

## 结构模板

```html
<h1>这里是文章标题</h1>

<section>
  <p>这里是摘要/导语。80-120 字，能独立理解。</p>
</section>

<p>开头段落。用冲突、误区、痛点或反常识引入。</p>

<h2>第一个小标题</h2>

<p>正文段落。手机屏幕上 2-4 句就该换段。</p>

<p>另起一段。<strong>重点句用 strong 标出来。</strong>每段最多一两处。</p>

<blockquote>特别关键的句子可以单独用 blockquote 突出。</blockquote>

<h2>第二个小标题</h2>

<p>继续正文。</p>

<h2>结尾</h2>

<p>总结 + 轻 CTA。不用「你值得」式的祝福结尾。</p>
```

## 段落规则

- 每个 `<p>` 2-4 句，手机屏幕上一个段落不要超过 5 行
- 段落之间空一行（默认 `<p>` 间距就够了）
- `<strong>` 标重点句，每段最多 1-2 处
- 小标题 `<h2>` 要有信息量——只看小标题就能知道文章骨架

## 配图占位块

默认不配图。如果用户要求配图，用文字占位，不用 `<img>` 本地路径：

```html
<p><strong>配图建议：</strong>这里放一张「提示词对比截图」，左边是模糊提问，右边是结构化提问，输出结果差异明显。</p>
```

## 摘要区

用 `<section>` 包起来，不指定任何样式：

```html
<section>
  <p>摘要正文，80-120 字。读者在信息流里能看到这段话。</p>
</section>
```

## 移动端阅读注意事项

- 公众号正文宽度约 375px
- 第一屏（滚动前）露出：封面区、标题、摘要、前 2-3 段
- 第一屏是唯一说服读者往下滚的机会
- 长文（3000 字以上）到中间应该有一个「为什么值得读完」的信号
- 篇幅建议：1500-3000 字。太短没信息量，太长手机阅读疲劳
